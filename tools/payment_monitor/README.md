# 收款进度监控工具

## 功能概述

从发票、合同、银行回执三种扫描件材料中提取信息，自动匹配并监控收款进度。

## 核心特性

- **本地 OCR 识别**：使用 PaddleOCR 本地处理，保护财务数据隐私
- **智能三方匹配**：基于客户名称相似度和金额容差的智能匹配算法
- **进度监控**：实时统计未收款、部分收款、已收款状态
- **Excel 报告**：自动生成统计摘要和明细列表

## 依赖安装

```bash
pip install paddleocr paddlepaddle pillow pymupdf openpyxl
```

如果需要 GPU 加速：
```bash
pip install paddlepaddle-gpu
```

## 使用方法

### 1. 准备扫描件

将扫描件按文件名分类：
- 合同：文件名包含"合同"或"contract"
- 发票：文件名包含"发票"或"invoice"  
- 回执：文件名包含"回执"、"receipt"或"银行"

支持格式：PDF、JPG、PNG

### 2. 配置参数

在 Web 界面的"配置"页面设置：
- **输入目录**：扫描件所在文件夹
- **OCR 语言**：中文（ch）或英文（en）
- **金额容差**：匹配时允许的金额差异百分比（默认 1%）
- **名称相似度阈值**：客户名称匹配的最低相似度（默认 0.8）

### 3. 启动 Web 界面

```powershell
py -c "import financial_analyzer_web as w; raise SystemExit(w.run_web(r'.\tools\payment_monitor\config.json', port=8765))"
```

### 4. 处理流程

1. 在"配置"页面设置参数并保存
2. 在主界面点击"开始"按钮
3. 查看"日志"页面监控处理进度
4. 处理完成后自动跳转到"查询结果"页面
5. 在"查询结果"页面查看统计和明细，可导出 Excel 报告

## 匹配逻辑

### 三方匹配规则

1. **合同-发票匹配**：
   - 客户名称相似度 >= 阈值
   - 金额差异 <= 容差

2. **发票-回执匹配**：
   - 付款方名称与客户名称相似度 >= 阈值
   - 金额差异 <= 容差

3. **收款状态计算**：
   - 未收款：收款比例 = 0
   - 部分收款：0 < 收款比例 < 1
   - 已收款：收款比例 >= 1

### 名称标准化

自动移除常见后缀：
- 有限公司、有限责任公司、股份有限公司
- Co.、Ltd.、Inc.

## 数据存储

- **数据库**：`data/warehouse.sqlite`
- **表结构**：
  - `contracts`：合同信息
  - `invoices`：发票信息
  - `receipts`：回执信息
  - `payment_matches`：匹配结果
  - `file_hashes`：文件去重

## 性能说明

- **CPU 版本**：2-5 秒/页（简单文档）
- **批量处理**：20-40 页/分钟（单线程）
- **文件去重**：自动跳过已处理的文件

## 注意事项

1. 首次运行需要下载 PaddleOCR 模型（约 10MB）
2. 扫描件质量影响识别准确率，建议使用 200 DPI 以上
3. 文件名需包含类型关键词（合同/发票/回执）以便自动分类
4. 正则表达式模式可在 `extractor.py` 中自定义扩展
