# 收款进度监控工具使用说明

## 功能概述

收款进度监控工具可以从发票、合同、银行回执三种扫描件中提取信息，并自动匹配判断收款状态。

## 使用流程

### 1. 配置参数

在"配置"页面设置以下参数：

- **输入目录**: 包含扫描件的文件夹路径
- **输出目录**: 报告输出目录
- **输出文件名**: 生成的 Excel 报告文件名
- **OCR 语言**: 选择中文或英文
- **使用 GPU**: 是否启用 GPU 加速（需要安装 GPU 版本的 PaddleOCR）
- **金额容差**: 匹配时允许的金额差异百分比（默认 1%）
- **名称相似度阈值**: 客户名称匹配的最低相似度（0-1，默认 0.8）
- **最大文件大小**: 单个文件的最大大小限制（MB）

### 2. 上传文档

在"上传文档"页面：

1. 点击上传区域或拖拽文件到上传区域
2. 支持的文件格式：PDF、JPG、PNG
3. 文件命名规则：
   - 合同文件：文件名包含"合同"或"contract"
   - 发票文件：文件名包含"发票"或"invoice"
   - 回执文件：文件名包含"回执"、"receipt"或"银行"
4. 点击"开始处理"按钮启动分析
5. 可以随时点击"取消"按钮停止处理

### 3. 查看数据看板

在"数据看板"页面可以查看：

- **统计卡片**: 总合同数、未收款、部分收款、已收款
- **金额统计**: 合同总金额、已收款总额、整体收款比例
- **收款状态分布图**: 饼图显示各状态的合同数量分布
- **Top 10 客户**: 柱状图显示合同金额和已收款金额对比
- **最近更新的合同**: 表格显示最近更新的合同明细

点击"刷新"按钮可以重新加载最新数据。

### 4. 查询结果

在"查询结果"页面可以：

- 按收款状态筛选（全部/未收款/部分收款/已收款）
- 按客户名称模糊搜索
- 查看详细的收款明细表格
- 点击"导出"按钮下载 Excel 报告

### 5. 查看处理结果

在"处理结果"页面可以查看：

- 处理结果摘要（处理文件数、匹配记录数）
- 生成的报告文件路径
- 数据库文件路径
- 处理日志和错误信息

## 数据匹配规则

### 三方匹配逻辑

1. **合同-发票匹配**:
   - 客户名称相似度 >= 阈值（默认 0.8）
   - 发票金额在合同金额的容差范围内（默认 ±1%）

2. **发票-回执匹配**:
   - 付款方名称与客户名称相似度 >= 阈值
   - 回执金额在发票金额的容差范围内

3. **收款状态判断**:
   - **未收款**: 收款比例 = 0%
   - **部分收款**: 0% < 收款比例 < 100%
   - **已收款**: 收款比例 >= 100%

### 支持场景

- ✅ 一个合同对应多张发票
- ✅ 一个合同对应多张回执
- ✅ 部分收款（允许分期付款）
- ✅ 超额收款（收款金额 > 合同金额）

## 注意事项

1. **文件命名**: 请确保文件名包含文档类型关键词（合同/发票/回执），否则无法自动识别
2. **OCR 性能**: CPU 版本处理速度约 2-5 秒/页，建议批量处理时耐心等待
3. **数据隐私**: 所有数据都在本地处理，不会上传到云端
4. **数据库**: 所有数据存储在 `data/warehouse.sqlite` 中，可以使用 SQLite 工具查看

## 依赖安装

如果遇到依赖缺失错误，请安装以下依赖：

```bash
# 基础依赖
pip install paddleocr pillow pymupdf openpyxl

# GPU 加速（可选）
pip install paddlepaddle-gpu
```

## 常见问题

**Q: OCR 识别速度慢怎么办？**
A: 可以尝试启用 GPU 加速，或者使用更小的图片分辨率。

**Q: 匹配结果不准确怎么办？**
A: 可以调整金额容差和名称相似度阈值，或者检查文件命名是否正确。

**Q: 如何查看历史数据？**
A: 所有数据都存储在数据库中，可以在"数据看板"和"查询结果"页面查看。

**Q: 如何导出报告？**
A: 在"查询结果"页面点击"导出"按钮，可以下载包含统计摘要和明细列表的 Excel 报告。
